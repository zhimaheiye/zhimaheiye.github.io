网络相关问题：

踩坑预警：Docker 代理配置导致的连接失败

在构建镜像时，可能会遇到类似 HTTP CONNECT: unexpected EOF xxx:443 的错误，导致拉取基础镜像失败。这通常是因为 Docker Desktop 的代理设置出现了问题。

解决办法：
如果你使用了 Clash 并且开启了 TUN 模式，可以直接把代理设置为本地回环地址。

  1. 打开 Docker Desktop 的 Settings -> Resources -> Proxies
  2. 开启 Manual proxy configuration
  3. HTTP 和 HTTPS 都填写：http://127.0.0.1:7897（端口号根据你的代理软件而定）
  4. Bypass 填写：localhost,127.0.0.1
  5. 点击 Apply & restart 重启 Docker 即可解决。

需求描述

想配置一套自动化的流程，自动抓取明日方舟的最新活动信息，并且放在我每天很容易看到的地方（对我来说就是每天都会创建的obsidian日程模板）。

解决思路：

关于抓取数据的渠道：
小红书、哔哩哔哩、官网等平台都会更新活动相关的信息，但是小红书哔哩哔哩等网站是需要登录校验的，而官网不需要登录，网页结构规范，所以选择了官网

(注：本脚本仅抓取官网公开数据，并在代码中加入了退避机制与合理延迟，严格遵守爬虫规范，仅作个人信息聚合使用。)

具体实现：
一开始是看到网络上有人使用 Open Router 等进行新闻的抓取，总结成一套 AI 早报，并且自己之前也了解过爬虫，认为用 AI 去抓取网络上的信息并分析是有可行性的。不过在跟 AI 讨论之后，判断官网展示的数据结构是较为规范的，可以通过写脚本来执行，不需要使用 AI。
